🎯 Problem

Cloud LLM APIs are expensive for experimentation and raise data privacy concerns for financial services use cases. When you're working with member data, credit risk metrics, or internal business logic, sending queries to OpenAI or Anthropic's API isn't always an option.

I wanted to test whether a 32B parameter reasoning model could run locally on consumer hardware with acceptable latency for real analytical tasks — SQL generation, financial reasoning, document analysis — without any data leaving the machine.

🔧 Approach

Set up DeepSeek-R1-Distill-Qwen-32B on an RTX 4090 (24GB VRAM) using quantized model formats (GGUF Q4_K_M, Q5_K_M). Benchmarked inference speed, VRAM usage, and output quality across different quantization levels and context lengths.

Tested with financial reasoning tasks like interpreting credit risk metrics, generating SQL queries from natural language descriptions, and analyzing regulatory compliance requirements — the kinds of tasks I actually encounter in my day job.

🧠 Key Decisions

Chose the 32B distilled variant over the full 671B model for VRAM feasibility. The 671B model requires 8x A100s or aggressive quantization that degrades quality significantly. The 32B distill retains most of the reasoning capability while fitting on a single consumer GPU.

Used llama.cpp for inference instead of vLLM because it handles quantized models better on single-GPU setups. vLLM is designed for multi-GPU serving workloads, while llama.cpp is optimized for single-card interactive use.

Tested multiple quantization levels to find the sweet spot between quality and speed. Q4_K_M vs Q5_K_M vs Q8_0 each have different VRAM/quality/speed tradeoffs that matter for different use cases.

Q4_K_M quantization fits in 20GB VRAM and generates ~15 tokens/sec — usable for interactive tasks. The model produces surprisingly strong SQL generation and financial reasoning outputs.

💻 Key Code

Model loading with quantization config and inference benchmarking loop.

inference.pyPython
from llama_cpp import Llama

llm = Llama(
    model_path="deepseek-r1-distill-qwen-32b-Q4_K_M.gguf",
    n_gpu_layers=-1,  # offload all layers to GPU
    n_ctx=4096,
    verbose=False
)
benchmark.pyPython
def benchmark(llm, prompt, max_tokens=512):
    t0 = time.perf_counter()
    out = llm.create_completion(
        prompt, max_tokens=max_tokens,
        temperature=0.7, top_p=0.9
    )
    elapsed = time.perf_counter() - t0
    tokens = out["usage"]["completion_tokens"]
    return tokens / elapsed  # tokens/sec
