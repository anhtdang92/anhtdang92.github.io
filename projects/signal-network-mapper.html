🎯 Problem

Military signal officers plan communications networks using static PowerPoint slides and manual diagrams. There's no interactive tooling for visualizing how signal nodes, relay points, and communication links connect across a tactical area of operations.

When you're planning comms for a battalion-level exercise — coordinating radio retransmission sites, satellite terminals, tactical operations centers, and fiber links — you need to see the network topology on real terrain, not on a whiteboard. You need to know if your retrans site has line-of-sight to the TOC, whether your link budget supports the distance, and what happens if a node goes down.

🔧 Approach

Built a web-based interactive mapper using JavaScript with geospatial libraries. Users place signal nodes (radio retransmission sites, TOCs, satellite terminals) on a real map and define communication links between them.

The tool automatically calculates line-of-sight, link budgets, and network topology. You can drag nodes to test different positions, add or remove links, and immediately see how the network responds. The visualization uses military-standard symbology so it's immediately readable by any signal officer.

🧠 Key Decisions

Built as a pure web app with no backend so it can run on any laptop in the field without internet connectivity. Military environments often have no internet access — or access is through a classified network that can't reach arbitrary web services. The app needs to work completely offline.

Used vanilla JavaScript to minimize dependencies and ensure it works in restricted browser environments common in military settings. Government laptops often run older browsers, block extensions, and can't install npm packages. Zero dependencies means zero compatibility issues.

Focused on Signal branch-specific workflows rather than trying to be a general-purpose mapping tool. Every feature is designed around how signal officers actually plan networks during FTXs and CPXs — not how a civilian GIS analyst would approach the problem.

💻 Key Code

Node placement logic and RF link budget calculation for line-of-sight analysis.

network.jsJavaScript
function addSignalNode(latlng, type) {
  const node = {
    id: crypto.randomUUID(),
    position: latlng,
    type: type,  // 'retrans' | 'toc' | 'satcom'
    links: [],
    status: 'active'
  };
  network.nodes.set(node.id, node);
  renderNode(node);
}
rf-calc.jsJavaScript
function calcLinkBudget(nodeA, nodeB, freqMHz) {
  const dist = haversine(nodeA.position, nodeB.position);
  const fspl = 20 * Math.log10(dist)
    + 20 * Math.log10(freqMHz) + 32.44;
  return { distance: dist, pathLoss: fspl };
}
