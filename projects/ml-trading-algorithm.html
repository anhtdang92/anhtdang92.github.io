🎯 Problem

Most retail trading strategies rely on single indicators or discretionary rules that don't generalize. RSI overbought, MACD crossover, moving average golden cross — these are lagging signals that every other market participant can see too.

I wanted to build a rigorous ML pipeline that treats trading as a classification problem — can a model learn to predict short-term price direction from engineered features better than random chance? And can it do so with proper validation that doesn't overfit to historical patterns?

🔧 Approach

Built a custom feature engineering pipeline generating 30+ technical and statistical features from OHLCV data. Features include standard technical indicators plus custom features like volume regime changes, volatility clustering metrics, and cross-timeframe momentum signals.

Implemented walk-forward backtesting to simulate real-world deployment — training on historical windows and testing on unseen future data to avoid overfitting. Used scikit-learn's gradient boosted classifiers with hyperparameter tuning via cross-validation.

🧠 Key Decisions

Walk-forward validation was non-negotiable — random train/test splits create lookahead bias in time-series data. If your model trains on data from next Tuesday to predict last Monday, your backtested Sharpe ratio is meaningless.

Used feature importance analysis to prune redundant indicators and prevent overfitting. Many technical indicators are correlated (RSI and Stochastic oscillator measure similar things), and including both just adds noise without signal.

Focused on risk-adjusted returns (Sharpe ratio) rather than raw accuracy as the optimization target. A 90% accuracy model that's wrong on the 10% biggest moves is worse than a 55% accuracy model that correctly captures volatility events.

def build_features(df):
    # Momentum & volatility regime features
    df['vol_regime'] = df['close'].pct_change().rolling(20).std()
    df['momentum_5'] = df['close'] / df['close'].shift(5) - 1
    df['vol_change'] = df['volume'] / df['volume'].rolling(10).mean()
    df['atr_14'] = compute_atr(df, period=14)

def walk_forward(X, y, window=252, step=21):
    results = []
    for start in range(0, len(X) - window, step):
        train = slice(start, start + window)
        test = slice(start + window, start + window + step)
        clf = GradientBoostingClassifier().fit(X[train], y[train])
        results.append(sharpe(clf, X[test], y[test]))
